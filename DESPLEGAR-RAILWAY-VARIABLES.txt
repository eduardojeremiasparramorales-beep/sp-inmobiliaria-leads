                    PROBLEMA ACTUAL
═════════════════════════════════════════════════════════════════════════════════

Meta rechaza la validación del webhook con error:
"No se pudo validar la URL de devolución de llamada o el token de verificación"

Causa: El archivo .env no se carga correctamente en Railway.

Solución: Agregar las variables de entorno directamente en Railway (en lugar de un archivo .env).

                    PASO 1 — IR A RAILWAY
═════════════════════════════════════════════════════════════════════════════════

1. Abre: https://railway.app
2. Inicia sesión con tu cuenta
3. Abre el proyecto: "sp-inmobiliaria-leads"
4. Click en el servicio principal (main o app)
5. Busca la sección "Variables" o "Environment"

═════════════════════════════════════════════════════════════════════════════════

═════════════════════════════════════════════════════════════════════════════════
                    PASO 4 — VALIDAR EL WEBHOOK EN META
═════════════════════════════════════════════════════════════════════════════════

Una vez que Railway esté actualizado:

1. Ve a: https://developers.facebook.com/apps/2533458103819028/whatsapp/
2. En "Webhook Configuration", haz click en "Verify and Save"
3. Meta intentará validar nuevamente
4. Debería decir ✓ "Verified" o "Validado"

Si aún falla, comprueba:
- Las variables están exactamente como arriba
- NODE_ENV=production (sin espacios)
- VERIFY_TOKEN=spInmobiliaria2026SecureVerifyToken (exacto)

                    ALTERNATIVA — VÍA RAILWAY CLI
═════════════════════════════════════════════════════════════════════════════════

Si prefieres usar la línea de comandos:

1. Instala Railway CLI: https://docs.railway.app/guides/cli
2. Abre terminal en C:\Sp Inmobiliaria\sp-inmobiliaria-leads-UPDATED\
3. Ejecuta:

   railway login
   railway link
   railway variables set NODE_ENV=production
   railway variables set VERIFY_TOKEN=spInmobiliaria2026SecureVerifyToken
