                    SI DECIDES USAR N8N
═════════════════════════════════════════════════════════════════════════════════

OPCIÓN 1 — N8N Cloud (Recomendado para empezar)
────────────────────────────────────────────────────────────────────────────────

1. Ve a: https://n8n.cloud
2. Crea una cuenta gratis
3. Crea un nuevo workflow
4. Busca los nodos de:
   - Webhook (para recibir datos)
   - HTTP Request (para enviar a Meta API)
   - Database (para guardar en DB)
   - WhatsApp (si existe)

Ventajas:
✓ Sin instalar nada
✓ Interfaz visual
✓ Hosting incluido
✓ Gratis hasta 100 execuciones/mes

Desventajas:
✗ Limitado en execuciones
✗ Puede ser más lento


OPCIÓN 2 — N8N Self-Hosted en Railway (Más control)
────────────────────────────────────────────────────────────────────────────────

1. Ve a: https://railway.app
2. Agrega nuevo servicio
3. Selecciona "n8n" del marketplace
4. Deploy automático
5. Accede en: https://tu-dominio.railway.app

Ventajas:
✓ Control total
✓ Sin límites de execuciones
✓ Conectado con tu BD

Desventajas:
✗ Necesita más recursos
✗ Más complejo de configurar


═════════════════════════════════════════════════════════════════════════════════
                    FLUJO N8N BÁSICO — RECIBIR Y RESPONDER
═════════════════════════════════════════════════════════════════════════════════

Si quisieras usar n8n (completamente opcional):

Nodo 1: Webhook
  └─ Escucha: POST /leads
  └─ Recibe datos de Meta

Nodo 2: Switch/If
  └─ Verifica si es nuevo lead
  └─ o respuesta de vendedor

Nodo 3A: Database
  └─ Guarda el lead en BD
  └─ Asigna a vendedor

Nodo 3B: HTTP Request
  └─ Envía respuesta a Meta API
  └─ Reenvía al cliente

Nodo 4: Slack (Opcional)
  └─ Notifica al equipo
  └─ "Nuevo lead recibido"
